Turnstile 防滥用配置

可为上传接口与后台登录启用 Cloudflare Turnstile 人机验证。

状态未知
启用后，上传接口会要求携带 Turnstile token，并在后端校验。
修改配置时需重新填写 Secret Key；为空将清空已有密钥。
「测试配置」将调用 Turnstile siteverify 验证当前 Secret Key 是否有效。
提示：仅在启用 Turnstile 时强制要求同时提供 Site Key 与 Secret Key。

新建存储桶

为单个存储桶填写完整的 R2 S3 配置与容量限制。

仅用于后台展示，便于区分不同用途的桶。
新建时必填；编辑时留空表示不修改现有密钥。
只写协议 + 域名，不要带路径或桶名，例如不要写 /bucket。
前端会换算为字节存入数据库；配额按 current_bytes + size <= max_bytes 校验。
关闭后该桶仍可通过固定 bucketID 上传，但不会被自动选中。
安全提示： R2 访问密钥仅存储在服务器数据库中，不会在前端回显。若怀疑泄露，可在 Cloudflare 后台重置密钥后在此处更新。
所有 R2 配置都从「存储桶管理」进入此表单，不再出现全局 R2 设置。
